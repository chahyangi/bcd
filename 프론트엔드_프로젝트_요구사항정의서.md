## 2. 기능 요구사항

기능을 우선순위별로 정리합니다. 우선순위는 아래 기준을 따릅니다.

- **P0 (필수)**: 없으면 프로젝트가 성립하지 않는 핵심 기능 : 블로그 기본 기능, 도형 및 색감 구현, id 정보에 도형 및 색감 지정
- **P1 (권장)**: 사용자 경험을 크게 향상시키는 기능 : id 정보에 따른 css 효과 구현
- **P2 (선택)**: 시간이 남으면 추가할 기능 : 한 쌍의 도형 및 색감 조합 5개 이상의 조합을 더한 비밀번호 구현

### 2-1. 기능 목록

| ID | 기능명 | 설명 | 우선순위 | 담당자 |
|----|--------|------|----------|--------|
| F-01 | 블로그 기본 구조 | 글 작성, 게시판 등 | P0 | 정다훈 |
| F-02 | 도형 구현 | 삼각형, 사각형 , 원 등의 기본도형 구현 | P0 | 정다훈  |
| F-03 | 도형 연계 아이디 구현 | 지정한 도형에 아이디 정보를 등록 | P0 | 정다훈 |
| F-04 | 도형 연계 css를 통한 디자인 구현 | 배경을 기본으로 지정하고 추가 기능 생각 | P1 | 정다훈 |
| F-05 | 도형 연계 비밀번호 구현 | 도형 개수 선택 및 도형 지정을 통해서 비밀번호 완성 조합 구상 | P2 | |

### 2-2. 기능 상세

#### F-01: 블로그 기본 구조 (P0)
- **설명**: 사용자가 글을 작성, 수정, 삭제하고 목록을 확인할 수 있는 기본적인 블로그 환경을 구축합니다.
- **사용자 시나리오**: 사용자가 메인 페이지에서 글 목록을 확인하고, 상세 페이지로 이동하거나 새 글을 작성합니다.
- **입력**: 게시글 제목, 본문(Markdown), 카테고리
- **출력/동작**: 게시글 리스트 렌더링, 상세 페이지 생성 및 조회
- **예외 처리**: 필수 입력 항목(제목, 본문) 누락 시 에러 메시지 노출

#### F-02: ID 연계 고유 도형 및 색상 생성 (P0)
- **설명**: 사용자 ID 문자열을 해싱(Hashing)하여 고유한 도형(삼각형, 사각형, 원 등)과 색상 값을 추출합니다.
- **사용자 시나리오**: 사용자가 회원가입 시 ID를 입력하면, 시스템이 해당 ID를 분석하여 고유한 '도형 ID'를 실시간으로 생성합니다.
- **입력**: 사용자 ID (String)
- **출력/동작**: ID 고유의 도형 형태(Type), 기본 색상(Hex), 테두리 곡률(Radius) 값 생성
- **예외 처리**: 중복 ID 입력 시 생성 프로세스 중단

#### F-03: ID 기반 동적 테마(CSS) 적용 (P1)
- **설명**: 유저 고유의 도형 정보를 기반으로 블로그의 전체적인 디자인 테마(CSS)를 자동으로 변경합니다.
- **사용자 시나리오**: 특정 유저의 블로그(`/[id]`)에 접속하면, 해당 유저의 고유 색상과 도형 스타일이 배경 및 버튼 UI에 적용됩니다.
- **입력**: 유저 고유 도형 속성 데이터
- **출력/동작**: CSS 변수(Variables) 업데이트를 통한 전역 스타일(배경색, 포인트 컬러, UI 곡률) 변경
- **비고**: Tailwind CSS의 동적 클래스 바인딩 활용

#### F-04: 도형 조합 비밀번호 인증 (P2)
- **설명**: 텍스트 대신 5개 이상의 도형과 색상 조합을 순서대로 선택하여 인증하는 시각적 보안 기능을 구현합니다.
- **사용자 시나리오**: 로그인 시 화면에 나타나는 도형 팔레트에서 자신이 설정한 순서대로 도형을 클릭하여 인증을 완료합니다.
- **입력**: 선택된 도형/색상 객체의 배열 (Array of Objects)
- **출력/동작**: 입력 조합과 저장된 조합 비교 후 인증 성공/실패 반환
- **예외 처리**: 인증 5회 실패 시 보안 코드 입력 또는 텍스트 비밀번호 전환 유도

## 3. 페이지 구조

### 3-1. 사이트맵

```
/ (홈) ------------------------- 전체 블로그 피드 및 최신 게시글 목록
├── /login -------------------- 일반 로그인 및 '도형 비밀번호' 입력 UI
├── /signup ------------------- 회원가입 (ID 입력 시 고유 도형 실시간 생성 프리뷰)
├── /about -------------------- 서비스 소개 (ID 기반 CSS 정체성 부여 로직 설명)
└── /user --------------------- 사용자 관련 공간
     ├── /[id] ---------------- 특정 유저의 블로그 (해당 ID의 도형/CSS 테마 적용)
     │    └── /post/[postId] -- 게시글 상세 페이지
     └── /settings ------------ 개인 설정
          ├── /profile -------- 내 고유 도형 및 정체성 확인
          └── /theme ---------- ID 기반 CSS 커스텀 (보조 색상 등 설정)
```

|-------------|----------|--------------|---------------|-------------|
| `/` | 홈 | Hero, CTA | X | |

### 4-2. 디자인 시스템

| 항목 | 값 | 비고 |
|:---|:---|:---|
| **Primary Color** | `rgb(var(--primary-rgb))` | 유저 ID 해싱을 통해 추출된 고유 RGB 값 |
| **Secondary Color** | `rgb(var(--secondary-rgb))` | Primary RGB 기반의 **Low-Light** (진한) 버전 |
| **Accent Color** | `rgb(var(--accent-rgb))` | Primary RGB 기반의 **High-Light** (연한) 버전 |
| **Font** | `Pretendard`, `sans-serif` | 가독성을 위한 산세리프 글꼴 |
| **Icon** | `Lucide React` | 도형 컨셉에 맞춘 깔끔한 라인 아이콘 |
| **Component Library** | `shadcn/ui` + `Tailwind CSS` | 동적 스타일링 및 일관된 UI 제공 |

#### 컬러 생성 규칙 (Color Logic)
1. **Primary**: 유저 ID 문자열을 0~255 사이의 R, G, B 값으로 변환하여 결정.
2. **Sub-Colors**: 
   - **Darker**: Primary RGB 값에서 각 요소를 20~30% 차감하여 구현 (Dark Mode 및 버튼 Hover용).
   - **Lighter**: Primary RGB 값에서 각 요소를 20~30% 가산하여 구현 (Background 및 Badge용).

## 5. 비기능 요구사항

| 항목 | 요구사항 상세 |
|:---|:---|
| **반응형 (Responsiveness)** | 데스크톱, 태블릿, 모바일 등 모든 기기 해상도에 대응하는 반응형 레이아웃을 구현하며, 특히 **도형 비밀번호 입력창**은 모바일 터치 환경에 최적화된 크기(최소 44x44px)를 유지함. |
| **브라우저 지원** | Chrome, Safari, Firefox, Edge 등 최신 웹 표준을 준수하는 **모던 브라우저**를 지원함. (IE 미지원) |
| **성능 (Performance)** | Next.js의 **SSR/SSG**를 활용하여 초기 로딩 속도(LCP) 1.5초 이내를 목표로 하며, ID 입력 시 실시간 도형 렌더링 시 버벅임이 없도록 최적화함. |
| **보안 (Security)** | 도형 조합 비밀번호는 단순 클라이언트 저장이 아닌, 서버 측 검증 로직을 거치며 데이터베이스 저장 시 **암호화(Hashing)** 처리를 필수 수행함. |
| **접근성 (Accessibility)** | 색상만으로 정보를 구분하기 어려운 사용자를 위해 **도형의 형태(삼각형, 원 등)**와 고유 아이콘을 병행 표기하고, 적절한 `aria-label` 속성을 부여함. |
| **사용자 경험 (UX)** | ID 입력에 따른 테마 변경 시 화면이 깜빡이는 현상(FOUC)을 방지하기 위해 CSS Variables 주입 방식을 최적화함. |
