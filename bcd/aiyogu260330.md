## 2. 기능 요구사항
| ID | 기능명 | 설명 | 상태 |
|----|--------|------|------|
| F-01 | Identity 로그인 | 3색 × 3모양 조합으로 ID 생성 및 입장 | 완료 |
| F-02 | 동적 테마(CSS) | ID 파싱을 통해 포인트 컬러 및 UI 형태 자동 적용 | 완료 |

| F-05 | 도형 비밀번호 | 5개 이상의 도형 조합을 통한 시각적 인증 | 대기 |

## 3. 페이지 구조

### 3-1. 사이트맵
/ (루트) ------------------------- 로그인 여부에 따른 자동 리다이렉트
├── /feed ----------------------- [Public] 전체 게시글 목록 (Global Feed)
├── /login ---------------------- [Public] 색상/모양 조합 선택 및 입장
├── /write ---------------------- [Private] 게시글 작성 (마크다운 지원)
└── /user ----------------------- 유저 관련 공간
    └── /[id] ------------------- 특정 유저의 개인 블로그 (ID 기반 테마 적용)
        └── /post/[postId] ------ 게시글 상세 페이지 (삭제 권한 로직 작동)

## 5. 비기능 요구사항
- **반응형**: 모든 기기 해상도 대응 및 모바일 최적화 UI.
- **성능**: router.replace 활용으로 불필요한 히스토리 적재 방지.
- **데이터 관리**: 서버 없이 localStorage를 활용한 데이터 영속성 유지.
- **보안**: 관리자 ID(red-triangle) 하드코딩을 통한 전역 관리 권한 제어.
